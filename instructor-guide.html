TechCamp Boot Camp · Managing student access · techcamp.live

§0 How the access system works

Students land on techcamp.live (index.html) and must enter their registered name and a personal access code you gave them. If both match, they're in. If not, they're blocked.

After 5 wrong attempts, they're locked out for 15 minutes. The certificate page also pre-fills their name from their login and won't let them change it — so they can only generate a certificate in their own name.

§1 Student pays on WhatsApp

Student texts you saying they want to register. You send your MoMo number. They send the payment screenshot. You confirm receipt.

§2 Give them a unique code

Invent a code for them. The format is TC-XXXX where XXXX is any 4 characters you choose. Keep a record of who has which code.

§3 Send them this WhatsApp message

Message template
Welcome to TechCamp!

Your portal is live at: techcamp.live

Your login details:
Name: [Their full name — exactly as registered]
Code: TC-XXXX

Enter these on the portal to access all your lessons, projects and the certificate generator. See you in class.
Important: The name must match what you register, but it is case-insensitive — "Kofi Mensah", "kofi mensah" and "KOFI MENSAH" all work. However "Kofi" alone without the surname will not.

§4 Add them to the student list

Open index.html in any text editor (Notepad on Windows, TextEdit on Mac, or VS Code). Find this section:

const STUDENTS = {
  // add your students here — name in lowercase, code uppercase
  "kwame asante":    "TC-KW24",
  "ama boateng":     "TC-AM97",
  // add new student on a new line here
  "new student name": "TC-XX99",
};

Add their name in lowercase and their code in UPPERCASE, with a comma after each entry. Save the file.

§5 Push to GitHub

1
Go to your GitHub repo at github.com/YOUR-USERNAME/techcamp-bootcamp
2
Click on index.html in the file list
3
Click the edit (pencil) icon to edit
4
Find the STUDENTS section and add the new student
5
Click Commit changes, then Commit changes again
6
Wait 1–2 minutes — techcamp.live updates automatically
You can also use the GitHub mobile app to edit files directly from your phone — so you can add a student within 2 minutes of receiving payment, even away from a computer.

§6 Removing a student (e.g. refund)

Delete their line from the STUDENTS list in index.html and push to GitHub. Their code stops working immediately, and their session expires when they close the browser tab.
