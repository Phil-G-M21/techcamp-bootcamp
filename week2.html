Microcontrollers
🧠 What is a Microcontroller?

A microcontroller is a tiny computer on a single chip. It contains a processor, memory, and programmable input/output pins — all in one. Unlike a PC that runs many programs, a microcontroller runs one program at a time, forever in a loop.

Microcontrollers are everywhere — inside your microwave, washing machine, TV remote, car dashboard, traffic lights, medical devices, and much more.

⚡
CPU Core
Processes your code instructions one by one.
💾
Flash Memory
Stores your program permanently even when powered off.
🧮
RAM
Temporary memory used while the program is running.
📌
I/O Pins
Connect to LEDs, sensors, motors, displays — the real world.

Breadboard
🧩 Understanding the Breadboard

A breadboard is a reusable plastic board with metal clips inside that let you build circuits by pushing component legs and wires into the holes — no soldering required. It's the most important prototyping tool in electronics.

Breadboard Internal Connections
+ Rail ──────────────────────────────── Power (+5V) runs along full length - Rail ──────────────────────────────── Ground runs along full length Rows (a–e and f–j) are connected horizontally: [ a ][ b ][ c ][ d ][ e ] ←── all 5 holes in same row are connected [ f ][ g ][ h ][ i ][ j ] ←── all 5 holes in same row are connected The center gap (between e and f) breaks the connection Columns run VERTICALLY in the power rails only
Power Rails
The red (+) and blue (–) long strips on the sides. Connect 5V and GND here to power your whole circuit.
Tie Points / Rows
The numbered rows in the middle. Each row of 5 holes is internally connected. Components sharing a row are electrically joined.
Center Gap
The gap in the middle separates the two halves. Perfect for placing ICs (chips) so each pin is isolated.
💡
Always connect your Arduino's 5V pin → red rail and GND pin → blue rail first. Then all your components can share power and ground from those rails easily.

C Language
💻 C Programming for Arduino — The Structure

Arduino code is written in C/C++. Unlike Python, C is a strongly-typed, compiled language — you must declare what type every variable is, and every statement must end with a semicolon ;

Every single Arduino program — no exceptions — must have exactly these two functions:

C / Arduino — the skeleton of EVERY program
// ── SETUP ──────────────────────────────────────────
// Runs ONE TIME only — when Arduino turns on or resets.
// Use this to: set pin modes, start Serial, configure sensors.
void setup() {

}

// ── LOOP ───────────────────────────────────────────
// Runs FOREVER in a continuous loop until power is cut.
// All your main program logic goes here.
void loop() {

}
void
Means the function returns nothing. setup() and loop() don't return any value — they just execute code.
setup()
Like plugging in and turning on a device. You configure everything once before the main program runs.
loop()
Like a heartbeat. Arduino keeps executing the code inside loop() over and over, thousands of times per second.
// comment
A line starting with // is a comment — Arduino ignores it. Write notes to yourself and others explaining your code.
🔄
setup() = getting dressed in the morning — once. loop() = breathing — never stops while you're alive (powered on). If you remove the power and reconnect, setup() runs again first.
Variables & Data Types
📦 Variables — Storing Data in C

A variable is a named container that stores a value in memory. In C — unlike Python — you must tell the compiler exactly what type of data it holds before using it. This is because the Arduino has very limited RAM (only 2KB on the Uno!) so the type determines how many bytes to reserve.

C — all data types explained
// ── INTEGER (int) ───────────────────────────────────
// Stores whole numbers. Range: -32,768 to 32,767
// Uses 2 bytes of RAM. Most common type in Arduino.
int ledPin   = 13;
int count    = 0;
int distance = -5;   // can be negative

// ── LONG ────────────────────────────────────────────
// Bigger whole numbers. Range: -2,147,483,648 to 2,147,483,647
// Use for millis(), big sensor values, timestamps.
long startTime = 0;
long duration  = 1500000;

// ── FLOAT ───────────────────────────────────────────
// Decimal numbers. Accurate to ~6-7 decimal places.
// Slower to process — use only when you need decimals.
float temperature = 36.5;
float voltage     = 3.14;

// ── BOOLEAN (bool) ──────────────────────────────────
// Only TWO possible values: true or false (1 or 0)
// Perfect for on/off, yes/no states.
bool ledOn    = true;
bool buttonHeld = false;

// ── BYTE ────────────────────────────────────────────
// Unsigned: 0 to 255. Uses only 1 byte of RAM.
// Great for brightness values, PWM, pin numbers.
byte brightness = 200;
byte speed      = 128;

// ── CHAR ────────────────────────────────────────────
// A single character. Stored as its ASCII number.
char grade  = 'A';

// ── STRING ──────────────────────────────────────────
// Text. Use sparingly on Arduino — eats RAM fast!
String name = "TechCamp";
⚠️
In C, every statement MUST end with a semicolon ; — missing it is the #1 beginner error. The compiler cannot find the next instruction without it and will refuse to compile.
pinMode()
📌 pinMode() — Configuring Pins

Before you can use any pin on the Arduino, you must tell it whether that pin will be sending signals (OUTPUT) or receiving signals (INPUT). This is done with pinMode() and it always goes in setup().

Think of each pin as an employee. pinMode() is the job description — you tell them whether they will be speaking (OUTPUT) or listening (INPUT).

C — pinMode() explained line by line
void setup() {

  // pinMode( PIN_NUMBER , MODE );
  //         ↑             ↑
  //         which pin     OUTPUT → pin sends voltage (5V or 0V)
  //                       INPUT  → pin reads voltage from outside
  //                       INPUT_PULLUP → reads, built-in pull-up resistor

  pinMode(13, OUTPUT);    // Pin 13 → sending (LED)
  pinMode(12, OUTPUT);    // Pin 12 → sending (another LED)
  pinMode(7,  INPUT);     // Pin 7  → receiving (button)
  pinMode(6,  INPUT_PULLUP); // Pin 6 → receiving w/ internal resistor
  pinMode(A0, INPUT);     // Analog A0 → receiving (potentiometer/sensor)

}
OUTPUT
The pin drives current out. Use for LEDs, buzzers, relays, motors (via driver). The Arduino controls what happens.
INPUT
The pin reads voltage coming in from outside. Use for buttons, sensors. Requires an external pull-down resistor to work reliably.
INPUT_PULLUP
Same as INPUT but activates Arduino's built-in 20kΩ pull-up resistor. Saves a physical resistor. Pin reads HIGH normally, LOW when button pressed.
⚠️
If you forget to call pinMode() before using a pin, Arduino defaults to INPUT mode — your OUTPUT will silently not work and the LED won't light. Always set pinMode first in setup()!
digitalWrite()
⚡ digitalWrite() — Turning Things ON and OFF

digitalWrite() controls digital OUTPUT pins. It sends either HIGH (5 volts) or LOW (0 volts) to a pin. That voltage change is what turns LEDs on/off, activates relays, sends signals to other devices.

Think of it as a light switch — HIGH = switch on, LOW = switch off.

C — digitalWrite() deep dive
// digitalWrite( PIN , VALUE );
//              ↑       ↑
//              pin     HIGH = send 5V (current flows → LED lights)
//                      LOW  = send 0V (no current  → LED goes dark)

void setup() {
  pinMode(13, OUTPUT);  // MUST set as OUTPUT first
}

void loop() {

  digitalWrite(13, HIGH);  // ← sends 5V → pin 13 goes HIGH → LED ON
  delay(1000);              // ← pause 1000ms (1 second)

  digitalWrite(13, LOW);   // ← sends 0V → pin 13 goes LOW → LED OFF
  delay(1000);              // ← pause 1 second before repeating

  // This loop runs forever → LED blinks endlessly
}

// You can also use HIGH=1 and LOW=0 as numbers:
// digitalWrite(13, 1);  // same as HIGH
// digitalWrite(13, 0);  // same as LOW

// Controlling multiple pins:
// digitalWrite(13, HIGH);  // LED 1 ON
// digitalWrite(12, LOW);   // LED 2 OFF
// Both happen almost simultaneously (microseconds apart)
⚡
digitalWrite() only works on digital pins (0–13) set as OUTPUT. Writing HIGH to a pin set as INPUT won't light an LED — it just enables the internal pull-up resistor. Always match pinMode and digitalWrite!
digitalRead()
👂 digitalRead() — Reading Buttons & Switches

digitalRead() reads the voltage on a digital INPUT pin. It returns either HIGH (pin sees ~5V) or LOW (pin sees 0V). Use this to detect button presses, switch states, or digital sensor signals.

C — digitalRead() with button — line by line
// ── Button Wiring ──────────────────────────────────
// One button leg → Pin 7
// Other button leg → GND
// 10kΩ resistor between Pin 7 and 5V (pull-down)

int buttonPin = 7;   // button connected to pin 7
int ledPin    = 13;  // LED on pin 13
int state     = 0;   // variable to store the reading

void setup() {
  pinMode(buttonPin, INPUT);   // button → reading incoming signal
  pinMode(ledPin, OUTPUT);     // LED → sending signal out
  Serial.begin(9600);          // open Serial Monitor
}

void loop() {

  state = digitalRead(buttonPin);
  // ↑ reads the voltage on pin 7
  //   returns HIGH (1) if button pressed
  //   returns LOW  (0) if button not pressed

  if (state == HIGH) {
    digitalWrite(ledPin, HIGH);        // button pressed → LED ON
    Serial.println("Button PRESSED"); // print to monitor
  } else {
    digitalWrite(ledPin, LOW);         // not pressed → LED OFF
    Serial.println("Button released");
  }

  delay(50);  // small delay to avoid reading too fast (debounce)
}

// ── With INPUT_PULLUP (no external resistor needed) ───
// pinMode(7, INPUT_PULLUP);
// → pin reads HIGH normally, LOW when button pressed
// → Logic is INVERTED: LOW = pressed, HIGH = released!
Pull-down Resistor
A 10kΩ resistor between the pin and GND. Ensures the pin reads LOW when button is open (not pressed). Prevents the pin from "floating" between HIGH and LOW randomly.
Floating Pin
An INPUT pin with nothing connected — it picks up electrical noise and reads random HIGH/LOW values. Always use a pull-up or pull-down resistor on input pins!
Debounce
Physical buttons "bounce" — they rapidly toggle HIGH/LOW several times in the milliseconds after being pressed. A small delay(50) or debounce code prevents reading false multiple presses.
analogRead()
〰️ analogRead() — Reading Sensors & Potentiometers

Digital pins are all-or-nothing — HIGH or LOW. But the real world has in-between values — a temperature isn't just "hot" or "cold", it's 27.5°C. analogRead() reads a voltage anywhere between 0V and 5V and converts it to a number from 0 to 1023 using the Arduino's built-in 10-bit ADC (Analog-to-Digital Converter).

C — analogRead() explained
// analogRead( PIN );
// Only works on analog pins: A0, A1, A2, A3, A4, A5
// Returns a number: 0 (0V) to 1023 (5V)
// 10-bit resolution → 2^10 = 1024 possible values

int sensorPin = A0;   // potentiometer/sensor on A0
int sensorValue = 0; // store the reading here

void setup() {
  Serial.begin(9600);
  // NOTE: analog pins don't need pinMode() for INPUT
  // They are INPUT by default — just read them directly
}

void loop() {

  sensorValue = analogRead(A0);
  // ↑ reads voltage on A0
  // 0V   → returns 0
  // 2.5V → returns 511
  // 5V   → returns 1023

  Serial.print("Raw value: ");
  Serial.println(sensorValue);    // e.g. 0 to 1023

  // ── Converting to Voltage ─────────────────────────
  // voltage = (sensorValue / 1023.0) × 5.0
  float voltage = (sensorValue / 1023.0) * 5.0;
  Serial.print("Voltage: ");
  Serial.print(voltage);
  Serial.println(" V");

  // ── Converting to Percentage (0-100%) ────────────
  int percent = map(sensorValue, 0, 1023, 0, 100);
  Serial.print("Level: ");
  Serial.print(percent);
  Serial.println("%");

  delay(500);
}
🔧
The map() function is your best friend for analogRead. It re-maps a value from one range to another: map(value, fromLow, fromHigh, toLow, toHigh). Use it to turn 0–1023 into 0–255 for PWM, or 0–100 for percentages.
analogWrite() & PWM
🌊 analogWrite() — Dimming LEDs & Speed Control

analogWrite() doesn't actually output a true analog voltage. Instead it uses PWM (Pulse Width Modulation) — it rapidly switches the pin between HIGH and LOW, so fast the LED appears to dim or the motor appears to slow down. The duty cycle (how long it stays HIGH vs LOW per cycle) controls the apparent power.

PWM Duty Cycle Visualised
Duty 0% (off): _______________________________ 0 brightness Duty 25%: ▌___▌___▌___▌___ dim Duty 50%: ▌▌__▌▌__▌▌__▌▌__ half brightness Duty 75%: ▌▌▌_▌▌▌_▌▌▌_▌▌▌_ bright Duty 100% (on): ▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌ full brightness
C — analogWrite() line by line
// analogWrite( PIN , VALUE );
//             ↑       ↑
//             pin     0   = fully OFF  (0% duty cycle)
//                     128 = half power (50% duty cycle)
//                     255 = fully ON   (100% duty cycle)
//
// ⚠ ONLY works on PWM pins: 3, 5, 6, 9, 10, 11
//   (marked with ~ tilde on Arduino Uno board)

int ledPin = 9;   // must be a PWM pin (~)

void setup() {
  pinMode(ledPin, OUTPUT);
}

void loop() {

  // ── Fade LED in (0% → 100%) ──────────────────────
  for (int brightness = 0; brightness <= 255; brightness++) {
    analogWrite(ledPin, brightness);
    // ↑ each iteration increases brightness by 1
    delay(8);  // small delay so fade is visible
  }

  // ── Fade LED out (100% → 0%) ─────────────────────
  for (int brightness = 255; brightness >= 0; brightness--) {
    analogWrite(ledPin, brightness);
    delay(8);
  }

  // ── Control motor speed from sensor ──────────────
  int sensorVal = analogRead(A0);       // 0 to 1023
  int speed     = map(sensorVal, 0, 1023, 0, 255);
  // ↑ converts sensor range to PWM range
  analogWrite(10, speed);  // motor speed follows sensor
}
delay() & millis()
⏱️ delay() & millis() — Controlling Time

delay() pauses the program for a number of milliseconds. It is the simplest timing tool — but it freezes the entire Arduino during the wait. While delay() is running, Arduino can't read sensors, check buttons, or do anything else.

millis() returns how many milliseconds have passed since Arduino turned on — without stopping anything. Think of it as checking a stopwatch while still running.

C — delay() vs millis() explained
// ── delay() — simple but BLOCKS everything ──────────
// delay( MILLISECONDS );
// 1000ms = 1 second, 500ms = 0.5 sec, 100ms = 0.1 sec

void loop() {
  digitalWrite(13, HIGH);
  delay(1000);   // ← Arduino FROZEN for 1 second
                  //   cannot read buttons, sensors etc.
  digitalWrite(13, LOW);
  delay(1000);   // ← frozen again
}

// ── millis() — non-blocking timing ──────────────────
// Returns unsigned long (time since Arduino started, in ms)
// Use when you need to do multiple things at the same time

unsigned long previousTime = 0;
int interval = 1000;   // blink every 1000ms
bool ledState = false;

void loop() {
  unsigned long currentTime = millis();
  // ↑ get current time — doesn't pause anything

  if (currentTime - previousTime >= interval) {
    // ↑ has 1000ms passed since last blink?
    previousTime = currentTime;   // save current time
    ledState = !ledState;         // toggle LED on/off
    digitalWrite(13, ledState);
  }

  // ↓ This still runs while LED is "waiting" to blink
  // Read sensor, check button — no freezing!
  int sensor = analogRead(A0);
}
⏱️
For beginner projects delay() is fine. Once you build more complex projects that need to do several things simultaneously (blink LED AND read sensor AND check button), switch to millis() for non-blocking timing.
Serial Communication
🖥️ Serial.begin() & Serial.print() — Talking to Your PC

The Serial port lets Arduino communicate with your computer over the USB cable — the same cable you use to upload code. This is your most powerful debugging tool. You can print sensor values, debug messages, variable values — anything — to the Serial Monitor in Arduino IDE.

C — all Serial functions explained
void setup() {
  Serial.begin(9600);
  // ↑ MUST call this first in setup() before any Serial use
  // 9600 = baud rate (bits per second)
  // Must match baud rate in Serial Monitor dropdown
  // Common: 9600, 115200. 9600 is fine for beginners.
}

void loop() {
  int temp = 27;
  float voltage = 3.14;

  // ── Serial.print() — prints WITHOUT newline ──────
  Serial.print("Temperature: ");   // cursor stays on same line
  Serial.print(temp);               // prints the number
  Serial.print(" degrees");

  // ── Serial.println() — prints WITH newline ────────
  Serial.println();                 // just moves to next line
  Serial.println(temp);             // prints number + newline
  Serial.println("Hello World");  // prints text + newline
  Serial.println(voltage, 2);      // float with 2 decimal places

  // ── Best practice — use labels ───────────────────
  Serial.print("Temp: ");   Serial.print(temp);
  Serial.print("  |  Voltage: ");   Serial.println(voltage);
  // Output: Temp: 27  |  Voltage: 3.14

  // ── Serial.available() — check for incoming data ──
  // Reads what YOU type in the Serial Monitor
  if (Serial.available() > 0) {
    char received = Serial.read();  // read one character
    Serial.print("You sent: ");
    Serial.println(received);
  }

  delay(1000);
}
🖥️
Open Serial Monitor: Tools → Serial Monitor (or Ctrl+Shift+M). Make sure the baud rate in the bottom dropdown matches your Serial.begin() number. If you see gibberish — the rates don't match!
Loops & Functions
🔁 Loops & Writing Your Own Functions

Loops repeat code automatically. Functions let you organise and reuse code — like creating your own Arduino commands.

C — loops and custom functions
// ── FOR LOOP — repeat exact number of times ──────────
// for ( start ; condition ; increment ) { }
//      ↑          ↑              ↑
//      int i=0   runs while     i++ means i = i + 1
for (int i = 0; i < 5; i++) {
  digitalWrite(13, HIGH); delay(200);
  digitalWrite(13, LOW);  delay(200);
  // blinks 5 times then stops
}

// ── WHILE LOOP — repeat while condition is true ───────
int count = 0;
while (count < 10) {
  Serial.println(count);
  count++;   // count = count + 1  (MUST update or infinite loop!)
}

// ── WRITING YOUR OWN FUNCTIONS ────────────────────────
// return_type functionName( parameters ) { body }
// void = returns nothing

void blinkLED(int pin, int times, int speed) {
  // ↑ custom function: which pin, how many blinks, delay
  for (int i = 0; i < times; i++) {
    digitalWrite(pin, HIGH); delay(speed);
    digitalWrite(pin, LOW);  delay(speed);
  }
}

int addNumbers(int a, int b) {
  // ↑ returns an int result
  return a + b;  // sends the value back to the caller
}

void setup() { pinMode(13, OUTPUT); }

void loop() {
  blinkLED(13, 3, 500);  // blink pin 13, 3 times, 500ms each
  blinkLED(13, 1, 100);  // blink pin 13, 1 time, fast
  int result = addNumbers(10, 25);
  Serial.println(result);  // prints 35
  delay(2000);
}
Quick Reference
📋 Complete Arduino Function Reference

Here's every key function you'll use — with exactly what each argument means:

pinMode(pin, mode)
pin = 0–13 or A0–A5. mode = OUTPUT, INPUT, or INPUT_PULLUP. Always call in setup() before using any pin.
digitalWrite(pin, val)
pin = digital pin 0–13. val = HIGH (5V, on) or LOW (0V, off). Pin must be OUTPUT.
digitalRead(pin)
pin = digital pin 0–13. Returns HIGH or LOW. Pin must be INPUT. Use for buttons/switches.
analogRead(pin)
pin = A0–A5 only. Returns 0–1023. No pinMode needed. Used for sensors, potentiometers.
analogWrite(pin, val)
pin = PWM pins ONLY: 3,5,6,9,10,11 (marked ~). val = 0 (off) to 255 (full). Controls brightness/speed via PWM.
delay(ms)
ms = milliseconds to pause. 1000 = 1 second. Freezes everything during wait.
millis()
Returns unsigned long — milliseconds since Arduino started. Never resets. Use for non-blocking timing.
Serial.begin(baud)
baud = communication speed. 9600 for beginners, 115200 for faster. Must match Serial Monitor setting.
Serial.print(val)
Prints val (text, number, variable) to Serial Monitor. No newline after. Cursor stays on same line.
Serial.println(val)
Same as print() but adds newline at end. Each println goes on its own line in Serial Monitor.
map(val,iL,iH,oL,oH)
Re-maps a number from input range to output range. e.g. map(512,0,1023,0,255) → 128. Essential for scaling sensor values.
constrain(val,min,max)
Clamps a value between min and max. e.g. constrain(300,0,255) → 255. Prevents values going out of range.

// LED Blink — TechCamp Week 2 Project 1

int ledPin = 13;   // LED on pin 13

void setup() {
  pinMode(ledPin, OUTPUT); // set pin 13 as output
}

void loop() {

📡 Project 2 — Ultrasonic Distance Sensor

The HC-SR04 is an ultrasonic distance sensor. It works like a bat — it sends out a high-frequency sound pulse (TRIG pin), which bounces off objects and returns (ECHO pin). By measuring the time it takes, we calculate the distance in centimetres.

This is the same principle used in car parking sensors, drones, robots and industrial automation.

📝 Ultrasonic Distance — Full Code
C / Arduino — HC-SR04 Distance ▶ Open Simulation on Wokwi 
// Ultrasonic Distance Sensor — TechCamp Week 2 Project 2

int trigPin = 9;
int echoPin = 10;

void setup() {
  Serial.begin(9600);
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
  Serial.println("Distance Sensor Ready!");
}

void loop() {
  // Step 1: Send trigger pulse
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);   // 10 microsecond pulse
  digitalWrite(trigPin, LOW);

  // Step 2: Measure echo duration
  long duration = pulseIn(echoPin, HIGH);

  // Step 3: Calculate distance
  // Speed of sound = 0.034 cm/µs
  // Divide by 2 because pulse travels there AND back
  int distance = duration * 0.034 / 2;

  // Step 4: Print to Serial Monitor
  Serial.print("Distance: ");
  Serial.print(distance);
  Serial.println(" cm");

  // Step 5: Danger alert with LED on pin 13
  if (distance < 10) {
    digitalWrite(13, HIGH);  // LED on — too close!
    Serial.println("⚠ WARNING: Object too close!");
  } else {
    digitalWrite(13, LOW);   // LED off — safe distance
  }

  delay(500);  // read every 0.5 seconds
}
delayMicroseconds(10)
Sends a 10 microsecond HIGH pulse on TRIG — this fires the ultrasonic burst.
pulseIn(echoPin, HIGH)
Measures how long ECHO stays HIGH in microseconds — this is the travel time.
duration * 0.034 / 2
Converts time to cm. Sound = 0.034 cm/µs. Divide by 2 (round trip).
if (distance < 10)
If closer than 10cm, turn on warning LED on pin 13.

Electronics · Fundamentals
⚡ Electricity Basics — Voltage, Current & Resistance

Before building any circuit, you need to understand the three fundamental properties of electricity. They work together in every circuit you will ever build.

⚡ Voltage (V) — the "pressure"
Voltage is the electrical force that pushes electrons through a circuit. Measured in Volts (V). Think of it like water pressure in a pipe — higher pressure pushes more water. Arduino operates at 5V. A USB port is 5V. A AA battery is 1.5V. A phone charger is typically 5V.
🌊 Current (I) — the "flow"
Current is the actual flow of electrons through the circuit. Measured in Amperes / Amps (A) or milliamps (mA). Think of it like the volume of water flowing through the pipe. An Arduino pin can safely supply a maximum of 40mA per pin. An LED needs about 20mA.
🧱 Resistance (R) — the "restriction"
Resistance opposes the flow of current. Measured in Ohms (Ω). Think of it like a narrow section of pipe slowing water flow. Resistors are components that add resistance on purpose — to protect LEDs, limit current, and divide voltage.
Electronics · Ohm's Law
🔺 Ohm's Law — The Most Important Formula in Electronics

Ohm's Law describes the relationship between Voltage, Current and Resistance. Know this formula and you can calculate anything in a basic circuit.

Ohm's Law Triangle — cover what you want to find
V ← Voltage (Volts) ───────── I × R ← Current × Resistance
V = I × R
Voltage = Current × Resistance. If you know current and resistance, calculate voltage.
I = V ÷ R
Current = Voltage ÷ Resistance. Used to find how much current flows. Most important for LED circuits!
R = V ÷ I
Resistance = Voltage ÷ Current. Used to calculate what resistor value to use.
// ── Practical Example: What resistor for an LED? ───────
//
// LED forward voltage (Vf)  = 2.0V  (red LED typical)
// Arduino pin voltage (Vs)  = 5.0V
// LED desired current (I)   = 20mA = 0.020A
//
// Voltage across resistor = Vs - Vf = 5.0 - 2.0 = 3.0V
// R = V ÷ I = 3.0 ÷ 0.020 = 150Ω
//
// → Use a 150Ω or the nearest standard value: 220Ω (safer)
// → Lower resistance = more current = brighter but may burn LED
// → Higher resistance = less current = dimmer but safer
💡
The most common beginner mistake is connecting an LED directly to 5V without a resistor. Without it, current is unlimited, the LED burns out instantly. Always use a resistor! When in doubt, 220Ω is safe for most LEDs at 5V.
Electronics · Resistors
🧱 Resistors — Reading Colour Codes

Resistors have colour bands painted on them that tell you their resistance value. Learning to read them means you can identify any resistor without measuring it. The most common are 4-band resistors.

4-Band Resistor Colour Code
Band 1 (1st digit) Band 2 (2nd digit) Band 3 (multiplier) Band 4 (tolerance) BLACK = 0 Brown = 1 Green = 5 Blue = 6 RED = 2 Orange= 3 Yellow= 4 Violet= 7 Grey = 8 White = 9 Multiplier: Black=×1 Brown=×10 Red=×100 Orange=×1000 Yellow=×10000 Tolerance: Gold=±5% Silver=±10% None=±20% Example: Red Red Brown Gold 2 2 ×10 ±5% = 220Ω ±5%
220Ω — Orange Red Brown
The most used resistor for LEDs with 5V Arduino. Limits current to ~15mA. Safe and bright.
1kΩ (1000Ω) — Brown Black Red
Used for buttons (pull-down), base resistors for transistors, general limiting.
10kΩ (10,000Ω) — Brown Black Orange
Standard pull-up/pull-down resistor for buttons and sensor data lines. Very common.
🔧
Can't read the colours? Use a multimeter set to Ohms (Ω) and touch the probes to both legs of the resistor. It will display the exact value. Or use the free Resistor Color Code Calculator at resistorguide.com.
Electronics · LEDs
💡 LEDs — Light Emitting Diodes

An LED (Light Emitting Diode) is a semiconductor component that emits light when current flows through it in the correct direction. Unlike a light bulb, LEDs are polarised — they only work one way round.

LED Anatomy & Polarity
┌─────────┐ │ LED │ Round dome │ │ └────┬────┘ │ ← Longer leg = ANODE (+) = connect to POSITIVE / resistor side │ ← Shorter leg = CATHODE (–) = connect to GND Also: flat side of the dome = Cathode (–)
🔴
Red LED
Forward voltage ~2.0V. Most common. 1.8–2.2V typical.
🟢
Green LED
Forward voltage ~2.1V. Similar to red.
🔵
Blue LED
Forward voltage ~3.2V. Needs a different resistor calculation.
⚪
White LED
Forward voltage ~3.2–3.4V. Brightest. Same as blue.
⚠️
If you connect an LED the wrong way round (backwards), it simply won't light up — it won't break. Just flip it around. However, connecting it without a resistor WILL burn it out permanently within seconds!
Electronics · Push Buttons
🔘 Push Buttons & Switches

A push button is a simple switch — it connects two points in a circuit when pressed and disconnects them when released. Buttons in starter kits are typically momentary tactile switches with 4 legs.

4-Leg Push Button — Internal Connections
A ──┤├── B ← A and B are connected internally (same side) │ │ switch switch ← when pressed, bridges A-B to C-D │ │ C ──┤├── D ← C and D are connected internally (same side) Simple wiring: One leg → Pin 7, Diagonal leg → GND Add 10kΩ between Pin 7 and 5V (pull-down resistor)
Momentary Switch
Connected only while held down. Releases back when you let go. Used in keyboards, game controllers, doorbells.
Latching Switch
Stays in its new position after pressing. Like a light wall switch. Less common in Arduino kits.
Debouncing
Physical buttons mechanically bounce 2–5ms when pressed. Arduino reads this as multiple presses. Fix with delay(50) or software debounce libraries.
Electronics · Capacitors
🔋 Capacitors — Storing & Smoothing Energy

A capacitor stores electrical charge and releases it quickly. Think of it like a small rechargeable battery that charges and discharges very fast. Capacitors are used to smooth out voltage spikes, filter noise in power lines, and create time delays in circuits.

🔵
Electrolytic
Polarised — has a + and – leg. The longer leg is +. Common in power supply filtering. e.g. 100µF, 1000µF.
🟡
Ceramic
Small, non-polarised (no polarity). Used for high-frequency noise filtering. e.g. 100nF (0.1µF).
📊
Measured in Farads
µF = microfarads (millionths). nF = nanofarads. pF = picofarads. Bigger = more charge stored.
⚡
Common Use
Place 100nF ceramic cap between 5V and GND near sensors to filter electrical noise from motors.
⚠️
Electrolytic capacitors are polarised — connect + to positive and – to negative. Reverse polarity can cause them to explode. The negative leg is usually marked with a white stripe on the side of the capacitor.
Electronics · Breadboard Deep Dive
🧩 Breadboard — Full Guide

The breadboard is your prototyping workspace. Understanding how connections work inside it is essential — the most common beginner mistakes come from misunderstanding the breadboard's internal connections.

Breadboard Internal Connections — Top View
+ + + + + + + + + + + + + + + ← Power rail (+) — full length horizontal - - - - - - - - - - - - - - - ← Ground rail (–) — full length horizontal a b c d e f g h i j Row 1 ← a-e connected, f-j connected, gap in middle a b c d e f g h i j Row 2 ← completely separate from row 1 a b c d e f g h i j Row 3 ↑ CENTER GAP — breaks connection between e and f ICs (chips) straddle this gap so each pin is isolated - - - - - - - - - - - - - - - ← Ground rail (–) + + + + + + + + + + + + + + + ← Power rail (+)
Power Rails (Red +, Blue –)
Run the full length horizontally. Connect Arduino 5V to red (+) and GND to blue (–). All components share power from these rails with short jumper wires.
Terminal Rows (numbered)
Each numbered row has TWO groups of 5 holes. Holes a-b-c-d-e in the same row number are all connected. f-g-h-i-j in the same row are also all connected to each other. But a-e and f-j are NOT connected (separated by the centre gap).
Centre Gap (DIP spacing)
The gap between columns e and f is exactly the right size for IC chips (like the 555 timer or ATmega). Place a chip across the gap so each leg is on its own isolated row.
Jumper Wires
Short wires that make connections on the breadboard. Male-to-male for breadboard-to-breadboard. Male-to-female for breadboard-to-Arduino pins. Colour code: Red=5V, Black/Brown=GND, other colours=signals.
Electronics · Components Guide
🛒 Components in Your Starter Kit — Explained

Here's every common component in a starter kit and exactly what it does:

🔴 LED (Light Emitting Diode)
Emits light when current flows through it. Polarised — longer leg (+). Always needs a current-limiting resistor (220Ω for 5V). Forward voltage: ~2V red/green, ~3.2V blue/white.
🧱 Resistor
Limits current flow. Value in Ohms (Ω). Read by colour bands. 220Ω for LEDs, 10kΩ for pull-ups, 1kΩ for general. Non-polarised — any direction.
🔵 Capacitor (Electrolytic)
Stores charge. Polarised — longer leg (+). Used for filtering power supply noise. Common values: 10µF, 100µF, 1000µF.
🔘 Push Button
Momentary switch. Connects circuit when pressed. 4 legs — A&B connected, C&D connected. Press bridges A/B to C/D. Needs pull-up or pull-down resistor.
🌀 Potentiometer
Variable resistor with a dial. 3 legs: left=GND, middle=signal (wiper), right=5V. Rotate to output 0–5V on the middle pin. Read with analogRead(). Great for controlling brightness/speed.
🔊 Buzzer (Active)
Makes a tone when 5V is applied. Active buzzers have a built-in oscillator — just power them. Passive buzzers need a PWM signal (analogWrite or tone()) to make different pitches.
🌡️ DHT11 Sensor
Measures temperature (0–50°C) and humidity (20–90%). Digital output — one data wire. Uses a single-wire protocol. Needs DHT library. Takes reading every 2 seconds minimum.
📡 HC-SR04 Ultrasonic
Measures distance 2cm–400cm using sound pulses. Two signal pins: TRIG (send) and ECHO (receive). Calculate: distance = duration × 0.034 / 2.
⚙️ DC Motor
Spins when current flows — direction depends on polarity. Cannot be powered directly from Arduino pin (too much current). Needs a motor driver like L298N or L293D. Speed via PWM.
🔁 Servo Motor
Rotates to a specific angle (0°–180°). 3 wires: Red=5V, Brown=GND, Orange=Signal. Controlled with the Servo library. servo.write(90) rotates to 90 degrees.
⚡ Transistor (NPN)
Electronic switch. 3 legs: Base (control), Collector (load), Emitter (GND). Small signal on Base controls large current through Collector. Used to switch motors, relays, high-power LEDs from Arduino's tiny pin current.
🔌 Relay Module
Electrically controlled switch. Lets Arduino switch 240V mains devices. Signal pin LOW=relay closes, HIGH=relay opens. Always isolate mains connections carefully!

C Console · Tips
💡 Console Tips
printf() vs cout
Both work. printf() is C style — use %d for int, %f for float, %s for string, %c for char, \n for newline. cout is C++ style — use << and endl.
Arduino Functions Won't Work
digitalWrite(), analogRead() etc. are Arduino-specific. In the console, simulate them with printf() showing what would happen: printf("Pin 13: HIGH\n");
Compiler Errors
If you see a red error — read it carefully. The line number is shown. Most errors: missing semicolons ;, wrong types, undeclared variables, mismatched braces {}.
Return 0
Every C main() function must end with return 0; — this tells the OS the program finished successfully. Forgetting it on some compilers causes a warning.
